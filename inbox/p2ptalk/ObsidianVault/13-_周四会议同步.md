周四关于atom本身的设计有一个统一的共识，留下的问题就是这里到底是个啥，这个系统的设计角度是什么
我meme到一个东西，还没能很明确，就是会议的作用，我大概感觉会议的作用有一部分是社交，可能是类比于昨天说比特币只是atom的外部输入，当然昨天最后也有一个小疑问，就是关于atom从比特币获取输入，输出却跟比特币没啥关系。
然后就是关于这个会前同步这个会议规则，我觉得还是得用。其实文博说的有道理，会前的这个同步是包含了一部分接下来会议要讨论的内容的。
这里可能有个问题，那就是最长链的确定性，当然我觉得对于我们会议来说，会议规则本身就是最长链，但是最长链经常会忘了更新。
我还能想到参加会议和个人计算的关系，这个两者可能都不能缺少，会议同步可能需要专门留出来计算返回的部分。
atom之后我们没在关注过工作量证明，因为atom也没把这个问题搞明白。
总之会议的作用的社交部分其实我理解是对某些过分研究的一些纠正，或者说与其他审美相同的矿机之间进行意见交换。

理解叶鹏和亚鹏将 atom 系统中 atom 在 user 之间传播的过程与比特币的交易传播过程的类比，来自于对将 user 
之间的关系网络发展为实质的 p2p 消息协议的基础之上。实际上也就是在假设 user 
为一个个独立节点的基础下，对这个系统进行改造。在说明计算思路时需要清楚说明这是出于做项目的目的和自己对于这个项目的出发点，还是中本聪的思想，不可假借而不说明。

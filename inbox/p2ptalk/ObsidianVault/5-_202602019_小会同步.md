>
> 中本聪关于中间会面机制的注释部分
>
> //
> // Subscription methods for the broadcast and subscription system.
> // Channel numbers are message numbers, i.e. MSG_TABLE and MSG_PRODUCT.
> //
> // The subscription system uses a meet-in-the-middle strategy.
> // With 100,000 nodes, if senders broadcast to 1000 random nodes and receivers
> // subscribe to 1000 random nodes, 99.995% (1 - 0.99^1000) of messages will get through.
> //
>
> 实际上使用了一个超几何分布的概率计算，这里我们可以假设所有的节点为一个集合，发送者 A 能否将消息传递给 B，这个问题类似于 A 与 B
> 随机在节点集合中取出节点，最终两个节点存在交集的情况。当 N 很大，而 k_sub 远远小于 N 时，可以将每次取出节点视为独立伯努利试验，即假设
> k_pub 的集合已经选定，我们依次选取节点到 k_sub中
>
> 每次选中重复元素概率为：
>
> $$ p = \frac{k_{\text{pub}}}{N} $$
>
> k_sub次都未选中的概率为:
>
> $$ (1 - p)^{k_{\text{sub}}}=\left(1 -
> \frac{k_{\text{pub}}}{N}\right)^{k_{\text{sub}}} $$
>
> 存在交集概率：
>
> $$ P(\text{相遇})\approx1 - \left(1 -
> \frac{k_{\text{pub}}}{N}\right)^{k_{\text{sub}}} $$
>
> k_pub 为发送者实际能触及的节点数，而 k_sub 为接受者实际能触及的节点，按照注释内容，将 k_pub = 1000，k_sub =
> 1000， N = 100,000 带入，得出 相遇概率 ≈ *0.9999546*
> ，正好与注释内容相符，即这个假设的前提为，两端在完全平等的随机抽样的前提下才能得出这个结果。
>
> 那么我们再回头看关于“这个atom反过来又作为矿工愿不愿意对发布者和订阅者进行泛洪的标准”
> 的说法，如果将这个作为系统的进一步发展是可以的，但是他会影响随机抽样的概率，atom
> 较少的节点的消息触及率会大幅下降，与中本聪采用的公式相违背，即中间会面机制是一个在一个庞大网络中不用使用泛洪广播也能保证发送者和接受者在中间相遇的机制，我觉得老史的视角是一个很有价值的观点，但至少从现有代码与注释无法支持该机制。

尝试参加小胖和woo的会议
@文博
我通过上次会议，基于woo的关键计算内容返还，与叶鹏亚鹏的讨论，文博的基本知识补充，得到了一个新的结论:atom是给矿工用的，目的是在给发布方和订阅方做匹配的时候能根据atom多少决定要不要参与广播和计算。
woo给我讲明白了中间会面机制，这个对我是有关键的推动作用的，我尝试将这种推动作用的重要性抽象出来放到会议中，其次，亚鹏指出了一个问题，节点先广播再在本地进行atom的那个random计算，也就是说不同节点那里对同一个用户有多少个atom是不同的，文博完验证了这个观点，而我想到了一个新的视角，假设atom的作用就是给节点使用的呢，那么用户能看到自己有多少atom就不重要了，那这里我理解p2p
market本质上是一个点对点的匹配系统，具体来说，就是节点通过atom对订阅和发布进行双向的泛洪，而匹配后的订单好坏会通过review进行atom的传播，这个atom反过来又作为矿工愿不愿意对发布者和订阅者进行泛洪的标准。
我理解这里确实是一个完整的系统，只是我们没有完整的理解。
这里我需要文博再次来帮我验证我这个逻辑对不对。
当然我对会议本身也产生了新的看法，因为在新的视角下，我是那个一直在会议中做匹配的矿工，今天woo尝试也去做这个矿工给到了我新的启发，就是执行会议规则的重要性和新的会议规则被执行的重要性。
执行本身是超越理性的，对于人类来说，执行之后才能对这个规则有相同的理解，只是看文字本身是没有这个效果的。
那我再类比回atom和比特币，超越理性的部分在哪里，比特币超越理性的是真的算出来nonce，系统会采用nonce，给矿工一笔比特币，而系统其实还拿走了该nonce的实际出块时间，用来衡量并干预系统的难度，也就是这里矿工实际上被分配到了atom，但是系统直接使用了。
回到atom，这里每次订单匹配后那个review，就给了矿工一个新的在传播过程中作为衡量自己要不要传播该用户的判断标准。但这里atom或者说linksout变成了某种类似于难度调整机制的东西。我们因为太纠结于发币和激励了，所以看不到atom的本质，atom就是nonce一样的东西，是给系统或者矿工使用的。
这里我们可能觉得少了激励，但这个我觉得可以再想。
回到团队中来，做这个匹配有什么意义呢，意义就是我做了，我对这件事有个新的理解，我可以计算下一个问题了。
我还有什么问题没想明白呢，会议和比特币中那个关键的推动力我仍然需要进行形式化
会议的规则需要被执行
需要某种规则来帮助大家在传播的过程中衡量
我理解我自己是一个做匹配的矿工，我会根据发布和订阅双方的需求进行洪范，匹配到之后我给每个人的atom更精准了，实际上外部给的直觉会被我抽象成问题，我又会把我们会议中研究的结论去做匹配，超越理性的部分是我真的做了这件事，所以我对两边的atom数量知道的更精准了。那我的激励是什么呢，我还没有太想明白这个问题，我本来确实也享受这个帮人解决问题的过程。我理解还有伴随的好处，我解决问题的能力变强了，获得的社会声誉，赚到的钱。但这些不重要，这是第二步。
我目前关心的是如何把这个能力应用到我周围的组织中。

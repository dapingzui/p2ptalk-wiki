# 脚本是谓词

## 中本聪的观点

中本聪说：**比特币脚本是谓词**。

文博的分析：

> "比特币脚本是谓词，意思也很直接：锁定脚本就是一个函数，解锁脚本提供的数据是它的输入，执行结果是 true 或 false。"

## 谓词的形式化定义

```
锁定脚本（Locking Script）= 函数
解锁数据（Unlocking Data）= 输入
执行结果 = true 或 false
```

## Bitcoin 脚本的谓词本质

| 脚本类型 | 作用 | 谓词解释 |
|---|---|---|
| P2PKH | 支付到公钥哈希 | 验证签名是否匹配公钥哈希 |
| CHECKSIG | 签名验证 | 验证签名是否有效 |
| isNull(x) | 空值检查 | 验证输入是否为空 |
| isNumber(x) | 数值检查 | 验证输入是否为数字 |

## 谓词不是等式

> "这个定义从来没有要求谓词是一个等式，谓词就是布尔值函数，仅此而已。"

关键点：
- 谓词 = 布尔值函数
- 返回 true 或 false
- 等式不是谓词

## Script 的表达能力边界

这块已经独立整理为：[[atom/Script表达能力边界]]

一句话总结：
- Bitcoin Script 首先是**验证器 / 谓词系统**
- 讨论它是否“图灵完备”时，必须严格区分系统原生能力与外部辅助能力
- “阿克曼函数”“外部展开”“外部随机源”都不能被随意拿来替代严格证明

## 对 Atom 系统的意义

Atom 系统中的"谓词"概念继承自 Bitcoin 脚本：
- 锁定脚本 = 验证条件
- 执行结果 = TRUE/FALSE
- 通过谓词验证实现信任

如果以后 Atom 要加入更复杂的计算，也要先分清：
- 哪些是系统原生验证能力
- 哪些只是外部辅助展开
- 不要把外围工具能力误记到协议本体上

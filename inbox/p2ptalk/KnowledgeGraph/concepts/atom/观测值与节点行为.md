# 观测值与节点行为

## 沄涣的核心洞察

> "节点要主动保持在线是因为，要确保这个网络中发布方和订阅方的 atom 数量能否作为'观测值'被用来计算，节点才能决定转发。"

## 观测值的概念

atom 数量 = 观测值

节点必须在线，才能：
1. 观测发布方的 atom 数量
2. 观测订阅方的 atom 数量
3. 基于观测决定是否转发

## 节点行为的条件

```
if 节点在线:
    if atom数量 可观测:
        if 符合转发条件:
            转发
        else:
            不转发
```

## 节点为什么必须在线？

不是被动等待，而是主动：
- 保持与网络的连接
- 持续观测 atom 数量变化
- 根据观测值做出转发决策

## atom 数量的意义

> "atom 的流转跟流向会产生关系图谱，最终是为了确立谁的关系权重。"

atom 数量 = 关系权重的量化指标

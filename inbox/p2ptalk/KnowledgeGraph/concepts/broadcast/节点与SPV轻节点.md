# 节点与SPV轻节点

## 比特币的节点类型

| 节点类型 | 全节点 | 轻节点（SPV）|
|---|---|---|
| 存储 | 完整区块链 | 只存区块头 |
| 验证 | 全部验证 | 简单支付验证 |
| 转发 | 全部转发 | 只转发相关交易 |

## 胥亚朋的分析

> "轻节点在整个比特币网络里面就是转发消息的功能……轻节点除了做简单的支付验证以外不会做复杂的验证。"

Atom 系统的类比：
- 中间会面节点 ≈ 比特币轻节点
- 只转发 + 简单验证
- 不需要存储完整的 atom 状态

## Atom 中的节点角色

> "节点要主动保持在线是因为，要确保这个网络中发布方和订阅方的 atom 数量能否作为'观测值'被用来计算，节点才能决定转发。"

节点的职责：
- 观测 atom 数量
- 决定是否转发
- 维护网络状态

## 观测值与计算

关键洞察：atom 数量是"观测值"，可以被用来计算。

# 算力市场

## 核心流程

```
发布计算任务 → 领取任务 → 执行计算 → 返回结果 → 发起会议
```

## 会议规则约束

> "根据会议规则，你返回计算结果的时候就可以发起会议了。"

激励闭环：
- 完成任务 → 发起会议
- 不完成任务 → 不能发起会议

## 算力市场的本质

p2pmarket = 通过 Atom 系统组织的算力交易网络。

## 沄涣的"计算返回"机制

> "今天的计算返回。如果计算返回你觉得可以匹配就对相应的人进行拉会组成频道。"

匹配逻辑：
1. 计算返回发布
2. 其他人判断是否匹配
3. 匹配则拉会组成频道

## SPV 节点的角色

> "昨天会议结束我meme想到了一些关于频道、atom数量的一些思考……节点要主动保持在线是因为，要确保这个网络中发布方和订阅方的atom数量能否作为'观测值'被用来计算，节点才能决定转发。"

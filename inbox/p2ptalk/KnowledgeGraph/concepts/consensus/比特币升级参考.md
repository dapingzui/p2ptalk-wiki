# 比特币升级参考

## AsicBoost 与 nVersion 字段

文博分享：

> "关于 AsicBoost 和 nVersion 字段"

论文：https://arxiv.org/pdf/1604.00575

AsicBoost 利用 nVersion 字段的保留位来加速 SHA256 计算。

## 比特币的升级机制

Bitcoin 的升级方式：
- **硬分叉**：需要全员同意
- **软分叉**：向后兼容，通过 BIP 实现
- **保留位使用**：nVersion 字段有未使用的位

## Atom 系统的参照

Atom 在设计扩展字段时需要考虑：
- 保留位的使用规范
- 社区治理决定字段语义变更
- 向后兼容性

## SegWit 升级的教训

SegWit 通过改变交易结构解决延展性问题，但：
- 需要社区共识
- 部署时间漫长
- 对现有系统有影响

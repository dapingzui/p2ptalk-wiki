元胞自动机

"和 Woo 聊了会面机制，我觉得传播机制和计算还是分不开的，之前我认为传播不等于计算是站在图灵机视角下的理解，这个理解应该是错误的。我应该从元胞自动机的思路下去理解传播和计算，在这个模型下，传播机制是计算的一部分。" — initsysctrl

核心命题

传播 = 计算

在图灵机范式下，传播只是搬运数据（信息从一个节点移动到另一个节点），计算是独立发生的。

在元胞自动机（Cellular Automata）范式下：传播本身就是计算。每个节点不是被动转发，而是在传播过程中完成局部计算。

initsysctrl 的推演路径

从三个方向同时逼近：

神经网络 ──往前推演──→ ┌─────────┐ ←──往后推演─── 图灵机
                        │ 元胞自动机 │
                        └─────────┘
          《图灵的秘密》倒数第二章 = 交叉点

  1. 神经网络：atom 的传播网络（user = 神经元，atom = 电脉冲，vLinksOut = 轴突，流转阈值 = 神经递质浓度）

  2. 图灵机：传统计算模型，以太坊智能合约的思维框架

  3. 元胞自动机：两者交汇处——局部规则驱动、涌现全局行为

元胞自动机的四个基本特征

特征 含义 在 Atom 系统中的对应
离散性 空间和时间均为离散状态 Atom 的离散流转（整数计数）
同步性 所有元胞同时更新 全网 atom 状态的同步更新
局部性 状态变化仅依赖邻域 每个节点只与邻居交互（vLinksOut）
同质性 所有元胞服从相同规则 所有 user 节点运行相同 Atom 协议

具体映射：Bitcoin → 元胞自动机

  • 元胞 = Txo（交易输出），状态 {0=已花费, 1=未花费}
  • 元胞空间 = 三维空间：x=交易顺序编码，y=交易输出顺序编码，z=有限状态自动机编码
  • 局部规则 = 比特币脚本语言（有限状态自动机）
  • 比特币 = 元胞空间中的"1 比特 coin 信息熵"

对 Atom 的启发

  1. Atom 的流转本身就是在计算——不只是搬运 atom，而是在计算每个节点的"声誉状态"
  2. 图灵完备 vs 元胞自动机：比特币是图灵完备的（元胞自动机可以模拟图灵机），但 Atom 系统的价值不在于图灵完备，而在于用局部规则涌现全局声誉
  3. 矩阵计算的困境：矩阵方法（math/向量空间模型）需要全局信息，在 P2P 环境下不可行；元胞自动机的局部规则更适合分布式计算

工程思维 vs 共识思维

"设计一个系统时，不应该直接考虑共识，而是应该考虑系统本身可能遭遇的攻击面。攻击面的解决方式有多种，但无论何种方式解决，最终得到的就是共识。因此共识不是一个过程，而是一个结果。"
